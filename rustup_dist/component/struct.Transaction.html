A Transaction tracks changes to the file system, allowing them to be rolled back in case of an error. Instead of deleting or overwriting file, the old copies are moved to a temporary folder. If the transaction is rolled back, they will be moved back into place. If the transaction is committed, these files are automatically cleaned up using the temp system.

All operations that create files will automatically create any intermediate directories in the path to the file if they do not already exist.

All operations that create files will fail if the destination already exists.
